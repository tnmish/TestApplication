# Тестовая часть функционала отправки рассылок

## 1. Структура
Для получения сообщений решила попробовать использовать REST Api, контроллер SendMessageController принимает сообщение 
```
http://localhost:5278/api/SendMessage/{UserID}
```
В теле запроса передаем сообщение с указанием темы
```
{
    "messageText": "Test message",
    "topic": {
        "id": 4,
        "name": "Topic4"
    }
}
```

Если указанный пользователь не наден, сервер возвращает ошибку Not Found 404. 
В противном случае сообщение отправляется по всем доступным каналам и от сервера приходит от вет OK (200)

Приложение использует EntityFramework для подключения к базе данных, помимо контекста приложения (ApplicationContext) был создан репозиторий для работы, где созданы все необходимые функции для взаимодействия с контекстом приложения.

Все настройки для подключения к базе данных, почтовому серверу и серверу Telegram лежат в файле config.json
